JMC Help and Commands

English Documentation (Adapted from MUME Wiki & Nerevar JMC)

Introduction

JMC (Jaba Mud Client) is a powerful MUD client with support for triggers, aliases, variables, and complex scripting. This document provides a summary of its core commands.

Commands

#action #alias #antisubstitute #autoreconnect #bell #char #connect #cr #echo #gag #group #highlight #hotkey #if #ignore #info #killall #log #loop #math #multiaction #path #read #showme #variable #zap

#action

Syntax: #action {input string} {actions} {priority}

This command allows you to program an action (or several) that must be performed when entering any text from the MUD. Triggers can capture words into variables %0-%9.

Example: #action {You are hungry.} {eat bread} {2}

#alias

Syntax: #alias {NAME} {COMMANDS}

Aliases allow you to shorten long commands. Variables %1-%9 represent words in the input line, and %0 represents the whole line.

Example: #alias {k} {kill %1}

#antisubstitute

Syntax: #antisubstitute {string portion}

Prevents #substitute and #gag from affecting lines containing the specified text.

#autoreconnect

Syntax: #autoreconnect [on|off]

Enables or disables automatic reconnection to the server.

#bell

Syntax: #bell

Produces the default system sound or PC speaker beep.

#char

Syntax: #char {SYMBOL}

Changes the command character (default is #).

#connect

Syntax: #connect <address> <port>

Establishes a connection to a MUD server.

#cr

Syntax: #cr

Sends a carriage return (ENTER) to the server.

#echo

Syntax: #echo

Toggles the display of commands executed by triggers.

#gag

Syntax: #gag {string portion}

Prevents lines containing the specified text from being displayed.

#group

Syntax: #group {list|delete|disable|enable|info|global|local} [group_name]

Manages groups of triggers or aliases.

#highlight

Syntax: #highlight {color} {string}

Highlights lines originating from the MUD containing the specified text with the chosen color.

Example: #highlight {light red, italic} {seriously wounded you}

#hotkey

Syntax: #hotkey {key name} {commands} {group}

Defines a keyboard shortcut.

Example: #hotkey f1 flee

#if

Syntax: #if {CONDITION} {COMMANDS}

Evaluates a condition and executes commands if the result is non-zero (true).

#ignore

Syntax: #ignore

Toggles the activation of all triggers.

#info

Syntax: #info

Displays counts of defined triggers, aliases, and variables.

#killall

Syntax: #killall

Removes all defined aliases, triggers, variables, and highlights.

#log

Syntax: #log {filename} {append|overwrite}

Starts logging session output to a file.

#loop

Syntax: #loop {from,to[:delay]} {commands}

Repeats commands in a loop, storing the counter in %0.

Example: #10 {sing} (Short form for 10 repetitions)

#math

Syntax: #math {variable} {expression}

Calculates a mathematical expression and stores it in a variable.

#multiaction

Syntax: #multiaction

Toggles whether multiple triggers can fire on the same line.

#path

Syntax: #path

Displays the current contents of the movement buffer.

#read

Syntax: #read {filename}

Loads and executes commands from a file.

#showme

Syntax: #showme {text}

Displays text on the local screen without sending it to the server.

#variable

Syntax: #variable {name} {value}

Creates a variable and assigns it a value. Use $name to reference it.

Example: #variable food bread

#zap

Syntax: #zap

Closes the current connection.
